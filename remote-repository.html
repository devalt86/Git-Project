Working with Remote Repositories

If you're the only person working on your project, you may never need to set up a remote repository. Developers use a remote repository when sharing data with teammates.

Remote repositories are versions of your project hosted on the internet or local network, accessible for all team members. The actual work on your project happens in your local repository, then those changes are uploaded to a remote repository to share; they are not usually used for working on files.

To clone a remote repository from GitHub to VS Code takes a few clicks.

  1. Click the ellipses (…) at the top of the VS Code side navigation.
  2. Hover over Remote.
  3. Click Add Remote.
  4. Copy and paste the shared repository's URL.

You can verify the remote repository is on your computer by typing the following commands in the VS Code Terminal:

  • git remote -v
  • origin https://github.com/OWNER/REPOSITORY.git (fetch)
  • origin https://github.com/OWNER/REPOSITORY.git (push)

The first one is the "fetch URL" for reading access. The second is the "push URL" to write data to the remote.

To add and manage files before pushing to the remote repository, you will need to switch to that repository's directory.

  1. Type cd [repository-name].
  2. Once you change the directory, type git branch to make sure you are on the main branch.
  3. Type git pull to receive all the latest changes from the GitHub server to your computer.
  4. From the main branch, type git checkout -b [naming convention] (ex. ashley/creating-a-how-to) to create a new branch so the main branch isn't affected.
    • It is best practice to have your name in the branch so your teammates know who made changes.

You are now ready to work on your files.

  1. Type git add . (all) or git add to add the files you want to commit.
  2. Type git commit -m [“your message as a regular sentence will be the title of the commit”] to commit your changes so you can push them the main branch.
  3. Type git pull --rebase origin main to pull the latest changes your teammates might have made.
  4. Type git push origin [name of branch] to push the files to the shared GitHub repository.
    • A pull request suggestion should pop up.
    • You can either copy the address or navigate to the shared repository's main page and the corresponding pull request banner should be displayed.
  5. Click Create a Pull Request.

If all is correct, you can click Squash and Merge, then Confirm Squash and Merge. There should be a message of Pull request successfully merged and closed.
