Basic Git Operations in VS Code

  1. In the VS Code menu bar, click Terminal
  2. Begin creating your html and CSS pages.
  3. Once you are ready to commit, click the Source Control icon.
  4. Click commit.
  5. When prompted, type git commit -m "first commit" (name your commit).
  6. Click the check mark in the upper-right of VS Code.
  7. You can also perform these basic Git operations in the Terminal box:
    • Status: git status displays the branch you are in and any files in red that have not been committed.
    • Add: git add displays in green the specific files you want to add to commit, or git add . to display all files to add to the commit.
    • Commit: git commit -m “your message as a regular sentence will be the title of the commit”. (If you are already on the main branch, this command updates the files in the GitHub repository.)
    • Pull: git pull gets all latest changes from GitHub server and puts on your computer. Make sure on master branch to pull any changes you may need to know about. With a pull request you are proposing that your changes in a separate branch should be merged (pulled in) with the main branch.
    • Push: git push origin pushes the files to the GitHub repository to merge with the main branch.

The command git push -u origin [branch] is useful when pushing a new branch. This creates an upstream tracking branch with a lasting relationship to your local branch.

Branching with VS Code

When you created your first repository in GitHub, the main branch was automatically created. You want to switch branches before beginning any changes to a project, so your main branch is not affected.

To do that, you can type git checkout -b [naming convention] (ex. git checkout -b ashley/creating-a-how-to). You can also click the ellipses (…) at the top of the VS Code side navigation, click Checkout to… > Create a new branch from… > type the command.

At this point, you can add new folders and files, or make changes to/update your files. If you are working on multiple tasks at once, you can create a new branch for each task and switch between branches so one does not affect the other, or the main branch.

Note:You want to commit changes in one branch before switching to another. This prevents tasks in one branch from being mixed in with tasks from another branch.
